## 2. Key Decisions

1. **Grid sizes: 4×4, 4×6 (default), 6×6.** The originally requested 5×5 default was dropped because 25 cells can't form pairs; agreed replacement is 4×6 (24 cards = 12 pairs).
2. **No asset files — everything procedural.** Card faces are drawn at runtime onto Phaser `CanvasTexture`s; symbols are emojis rendered as text via the OS emoji font (Segoe UI Emoji / Apple Color Emoji / Noto Color Emoji). No font or image loading needed.
3. **Card backs:** a static set of 8 contrasting two-color pairs (`src/gradients.ts`); one pair is picked randomly at game start and the resulting linear-gradient texture is shared by all cards.
4. **Card fronts:** 6 noise variants (pastel diagonal gradient + speckle noise), generated once in `PreloadScene` and assigned randomly per card. The emoji symbol is a separate text object on top, so fronts differ visually even within a pair — matching is by symbol only.
5. **Cards are portrait rectangles with rounded corners.** Textures are generated at a fixed base resolution (200×270, radius 24) and scaled to fit the grid, so texture generation is independent of grid size.
6. **Players: 2–4, chosen in settings; all human for now.** `Player.type` is `'human' | 'computer'` — the type exists specifically as the hook for future computer opponents.
7. **Settings persistence:** localStorage (`memory-phaser-settings`), with validation/fallback to defaults on corrupt data.
8. **Skip rule interpretation:** "Skip turn" is only available before the first card of a pick is flipped. Once the first card is up, the player must complete the pick.
9. **Matched cards stay on the board:** face-up, dimmed, framed in the scoring player's color (doubles as a score visualization).
10. **Fixed virtual resolution 1024×768** with `Phaser.Scale.FIT` + center; page-level centering via CSS grid in `index.html`.

## 3. Game Flow (implemented rules)

1. All cards are revealed for **5 seconds** with a countdown, then flip face-down (staggered animation).
2. Start player is chosen **randomly**.
3. The active player has exactly two options: **pick cards** or **skip the turn**.
4. Match → pair stays revealed (framed in player color), score +1, same player goes again.
   Mismatch → both cards shown ~900 ms, flipped back, turn passes to the next player.
5. Game ends when all pairs are found. **Most pairs wins**; ties are reported as a draw. End overlay shows the full ranking with Play Again / Main Menu.

### GameScene state machine

`memorize` → `turn-idle` (pick or skip) → `turn-pick` (first card up, must finish) → `resolving` → back to `turn-idle` (match) or next player's `turn-idle` (mismatch) → … → `gameover`.

All input is guarded by this phase plus per-card flags (`matched`, `isFaceUp`, `isFlipping`), so double-clicks and clicks during animations are ignored.

## 5. Extension Points / Next Steps

- **Computer opponents** (planned): implement a controller invoked from `GameScene.startTurn()` when `player.type === 'computer'`. It should act through the same entry points humans use (`onCardClicked(card)` / `onSkip()`). A natural design: the AI keeps a memory of revealed cards with a configurable forget-rate for difficulty levels.
- **Real assets:** `PreloadScene.preload()` already wires a progress bar; drop `this.load.*` calls there.
- **Player naming/types in settings** once AI exists (e.g. per-slot Human/CPU toggle).
- Possible polish: sounds, flip-all animation juice, bundle size (Phaser is ~1.5 MB minified; Vite warns — ignorable or solvable via manual chunks).

## 6. Known Constraints

- Emoji appearance varies per OS/font; on Windows the symbols are color glyphs via Segoe UI Emoji.
- "Exit" in the main menu calls `window.close()`, which browsers block for tabs the script didn't open; it falls back to a "close the tab" hint.
- Build output was only verified via `tsc` + `vite build` and a click-through by the developer; no automated tests yet.
